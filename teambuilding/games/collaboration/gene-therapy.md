# 基因疗法攻坚 (Gene Therapy Sprint)

> **团队协作 · 科研模拟 · 压力决策**  
> 适合：团队协作 / 问题解决 / 创新思维训练

---

## 🎯 游戏目标

模拟基因疗法研发的紧急攻关过程，让团队在**时间压力**和**资源限制**下体验**协作研发、信息共享、风险决策**的重要性。

**团建核心价值**：
- 🤝 **团队协作**：必须分工合作才能完成任务
- 💬 **沟通表达**：科研信息分享和方案讨论
- 🎯 **问题解决**：在有限资源下优化研发路径
- 💡 **创新思维**：尝试不同的基因编辑策略
- ⏰ **压力管理**：在病毒扩散的时钟下保持冷静

## 🎮 游戏流程

### 准备阶段（5分钟）

1. **讲解背景**：罕见病患者等待基因疗法救命，团队需要在病毒扩散前完成研发
2. **选择角色**：每位玩家选择1个科研角色
3. **布置实验室**：放置地图，设置初始病毒进度（0）和科研进度（0/5）
4. **分发起始资源**：每人获得2张基因序列卡和1张编辑工具卡

### 游戏回合（每回合10-15分钟）

每回合分为4个阶段：

#### 阶段1：团队讨论（3分钟）

**关键决策**：
- 本回合优先推进哪个研发环节？
- 谁去哪个功能区执行行动？
- 基因序列卡和编辑工具卡如何分配？
- 如何应对可能的实验室事件？

**教练观察点**：
- 谁在主导技术讨论？
- 是否充分讨论不同编辑方案？
- 决策过程是否科学民主？
- 有没有明确的技术路线？

#### 阶段2：执行行动（5-7分钟）

每位玩家依次执行行动（每人每回合3个行动点）：

| 行动 | 消耗 | 效果 | 团建要素 |
| :--- | :--- | :--- | :--- |
| 🧬 收集基因 | 1点 | 抽取1张基因序列卡 | 信息收集 |
| ✂️ 设计编辑 | 2点 | 支付成本，修复1个突变基因 | 技术应用 |
| 🧪 安全测试 | 1点 | 对已修复基因进行安全性测试 | 风险管理 |
| 📊 分析数据 | 1点 | 查看当前进度，调整策略 | 数据分析 |
| 💉 临床试验 | 2点 | 推进临床试验进度1格 | 实践验证 |
| 🔄 资源交换 | 1点 | 与队友交换1张卡牌 | 协作共享 |
| 📋 项目协调 | 1点 | 重新分配资源优先级 | 项目管理 |

#### 阶段3：病毒扩散（2-3分钟）

1. **病毒进度推进**：翻开1张病毒卡，病毒进度+1（若病毒进度达到10，游戏失败！）
2. **实验室事件**：翻开1张事件卡
   - ✅ 好消息：获得额外资源 或 科研进度+1
   - ❌ 坏消息：丢失手牌 或 病毒进度+2
   - ⚡ 实验意外：需要进行额外安全测试
   - 🌈 技术突破：可立即完成1个研发环节

**关键机制**：
- 病毒进度是共享的，团队共同面对时间压力
- 事件卡增加不确定性和应变需求

#### 阶段4：简短复盘（1-2分钟）

每回合结束后，教练可提问：
- 本回合的技术路线是否最优？
- 有没有更好的基因编辑方案？
- 团队协作是否顺畅？
- 如何应对加速扩散的病毒？

## 🏆 胜利与失败

### 胜利条件

✅ **团队胜利**：完成以下5个研发环节！

**研发进度追踪**（需要按顺序完成）：
1. 🧬 **基因测序** - 收集3组正常基因序列 + 2组突变基因序列
2. ✂️ **编辑工具设计** - 成功使用3张编辑工具卡修复突变
3. 🧪 **安全测试** - 通过3次安全性测试（掷骰判定）
4. 💉 **临床试验** - 推进临床试验进度达到3格
5. 📋 **疗法批准** - 项目管理者宣布疗法 ready，全员投票通过

### 失败条件

❌ **团队失败**（任一发生）：
- 病毒进度达到10（病毒全面扩散）
- 安全测试连续失败3次（疗法存在重大安全隐患）
- 回合数达到8回合仍未完成研发（时间耗尽）

| 阶段 | 时长 | 内容 |
| :--- | :--- | :--- |
| 游戏讲解 | 5分钟 | 规则说明、角色介绍、科研背景 |
| 团队组建 | 5分钟 | 选择角色、讨论研发策略 |

| 团队复盘 | 10分钟 | 总结经验、分享科研感悟 |

### 快速流程（45分钟）

若时间有限，可简化：
- 研发环节缩减为3个（基因测序、编辑工具、安全测试）
- 病毒进度改为6即失败
- 进行3回合即揭晓结果

## 💡 教练引导指南

### 游戏前（设定期望）

**开场话术**：
> "接下来我们要进行一场基因疗法的紧急攻关。你们是一个顶尖科研团队的成员，必须在病毒全面扩散前完成基因疗法的研发。这个游戏模拟了真实的科研协作过程——你们需要收集数据、设计技术方案、测试安全性、推进临床试验。关键在于——你们能否高效协作、充分发挥各自的专业优势、在时间压力下做出科学决策。"

**强调要点**：
- 这是合作游戏，模拟科研协作
- 专业分工很重要，每个角色都不可或缺
- 信息共享和方案讨论至关重要
- 失败是科研的一部分，重要的是从失败中学习

### 游戏中（观察与引导）

**观察要点**：

1. **技术讨论质量**：
   - 是否充分讨论不同基因编辑方案？
   - 有没有基于证据的科学讨论？
   - 遇到技术难题时，团队如何应对？

2. **协作模式**：
   - 角色之间如何配合？
   - 资源（卡牌）是否共享？
   - 有没有出现"技术孤岛"？

3. **决策过程**：
   - 技术决策是如何做出的？
   - 是否权衡了风险和收益？
   - 面对不确定性，如何决策？

4. **压力应对**：
   - 病毒进度加速时，团队情绪如何？
   - 有人保持冷静并提出解决方案吗？
   - 压力下决策质量如何？

**适时介入**：
- 若团队陷入技术争论：提醒"时间紧迫，需要基于现有证据做决策"
- 若有人被技术讨论边缘化：提问"XX，作为[角色]，你怎么看这个方案？"
- 若资源管理混乱：建议"项目经理，你需要协调一下资源分配"

### 游戏后（深度复盘）

**复盘框架**（使用 toolbox/debrief.md）：

#### 1. 事实回顾（5分钟）

**提问**：
- 游戏结果是什么？（研发成功/病毒扩散/时间耗尽）
- 你们完成了几个研发环节？
- 哪个研发环节最耗时？为什么？

#### 2. 感受分享（5分钟）

**轮流发言**：
- 你在游戏中扮演的角色让你有什么感受？
- 哪个时刻让你感到最有压力/最有成就感？
- 你对团队的研发协作满意吗？

#### 3. 学习洞察（10分钟）

**讨论问题**：

**关于科研协作**：
- 你们的技术讨论是否充分？
- 不同角色的专业优势是否充分发挥？
- 有没有出现"各自为战"的情况？

**关于风险管理**：
- 你们如何平衡研发速度和安全性？
- 面对安全测试失败，团队如何应对？
- 在不确定情况下，如何做决策？

**关于资源管理**：
- 基因序列卡和编辑工具卡分配是否合理？
- 有没有出现资源浪费或资源不足？
- 项目经理的协调作用发挥如何？

**关于压力应对**：
- 病毒进度加速时，团队情绪和决策有何变化？
- 如何在时间压力下保持理性思考？
- 压力对团队协作有何影响？

#### 4. 行动迁移（5分钟）

**联系实际工作**：
- 这个游戏让你想到研发工作中的哪些场景？
- 游戏中的协作经验如何应用到实际项目中？
- 你们团队可以立即改进的研发协作方式是什么？

## 🔄 变体玩法

### 变体1：跨学科挑战（沟通训练）

**规则**：
- 每个角色只能用自己专业的术语解释方案
- 其他角色必须"翻译"成通俗语言才能执行
- 模拟跨学科沟通的挑战

**目的**：
- 体验专业术语造成的沟通障碍
- 锻炼"翻译"和解释能力
- 培养跨领域协作意识

**复盘重点**：
- 专业壁垒如何影响协作？
- 如何有效进行跨专业沟通？
- 通俗化表达的重要性

### 变体2：资源受限模式（创新思维）

**规则**：
- 起始手牌减半（每人只有1张基因序列卡和1张编辑工具卡）
- 必须通过协作和交换才能完成任务
- 强制创新：用有限资源找到解决方案

**目的**：
- 训练资源受限下的创新能力
- 强化协作和资源共享意识
- 体验"少即是多"的创新思维

### 变体3：伦理困境（价值观讨论）

**规则**：
- 游戏中加入"伦理决策卡"
- 例如：是否使用有争议的基因编辑技术？是否进行高风险的人体试验？
- 团队必须全员一致通过才能执行争议方案

**目的**：
- 引发科研伦理讨论
- 体验价值观冲突和共识达成
- 培养负责任的研究态度

**复盘重点**：
- 伦理考量在技术决策中的权重
- 如何平衡科学进步和伦理风险
- 团队价值观冲突如何解决

### 变体4：竞速模式（压力测试）

**规则**：
- 设置严格的时间限制（如30分钟）
- 病毒进度加速（每回合+2）
- 强制快速决策，减少讨论时间

**目的**：
- 体验高压研发环境
- 训练快速共识达成
- 观察时间压力下的团队协作

## 📝 常见问题

**Q：如果团队一直无法完成研发怎么办？**
A：可以降低难度（病毒进度每回合+0.5，或研发环节减少）。重点是体验科研协作过程，不是结果。复盘时讨论"如果重新来，会如何改进研发策略"。

**Q：有人过于坚持自己的技术方案怎么办？**
A：适时介入，提醒"科研需要开放讨论和证据支持"。复盘时讨论"如何平衡专业自信和开放心态"。

**Q：有人对基因编辑等科学内容不熟悉怎么办？**
A：游戏设计为科普级，不需要深奥知识。可以提供"科研速查卡"帮助理解。重点是协作过程，不是专业技术。

**Q：游戏时间拖得太长怎么办？**
A：设置严格的时间限制，用计时器。或简化为"快速流程"（3个研发环节）。

**Q：能不能多人扮演同一个角色？**
A：可以，但建议4-6人最佳。超过6人会有等待时间，影响体验。可以合并某些角色（如2人共同担任"编辑工具专家"）。
