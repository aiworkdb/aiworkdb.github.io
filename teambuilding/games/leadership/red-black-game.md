# 红黑游戏（Red-Black Game）

> 领导力培训经典游戏。两个小组在信息不对称下反复决策，体验信任、竞争与合作的根本张力。

---

## 1. 概览与标签

| 项目 | 内容 |
|------|------|
| **游戏名称** | 红黑游戏（Red-Black Game） |
| **目的标签** | 领导力 · 信任建立 · 问题解决 |
| **形式标签** | 室内 · 12–40人 · 智力 |
| **人数** | 12–40 人（最佳 16–30 人） |
| **时长** | 中篇 45–60 min（含深度复盘） |
| **难度** | ⭐⭐⭐ 中等偏难（对教练引导要求高） |
| **场景标签** | 培训 · 管理层工作坊 |
| **核心目标** | 体验竞争vs合作、信任建立、跨部门协作的复杂性 |

---

## 2. 核心规则

### 准备

- 将参与者分成 **A、B 两个小组**，安排在**不同房间**（或同一房间但背对背，不能看到对方的讨论）
- 准备 **得分矩阵**（PPT 投影 或 白板书写）

### 得分矩阵（每轮同时公布）

```
        B组选
        红    │    黑
A   ───────────────────────
组  红  A:-3  B:-3  │  A:+5  B:-5
选  黑  A:-5  B:+5  │  A:+3  B:+3
```

> **解读**：
> - 双方都选**黑**：各得 **+3 分**（合作共赢）
> - 双方都选**红**：各得 **-3 分**（互相伤害）
> - 一方**红**、一方**黑**：红方 **+5**，黑方 **-5**（背叛获利）

### 流程

```
第一轮：
1. A、B 两组分别进入不同房间
2. 每组内部讨论 3 分钟，决定本轮选"红"还是"黑"
3. 讨论结束后，两组代表同时公布选择（教练协调）
4. 教练按得分矩阵计算并公布两组得分

第二轮至第六轮：
5. 重复上述流程，共进行 5–7 轮
6. 每轮之间，允许两组各派 1 名代表"谈判"（1 分钟）
   → 谈判中可以说任何话，但没有约束力

结束后：
7. 两组回到同一空间，公布总分
8. 进入深度复盘（至少 20 分钟）
```

### 胜负判定

- **合作目标**：两组总分之和最高（理论最高 = 每轮 +6 × 轮数）
- **竞争目标**：自己组分数 > 对方组分数
- **复盘重点**：两组采取了什么策略？为什么？

## 3. 引导要点

### 如何讲解（3 分钟讲完）

```
（两组集合在一起时讲解）

"大家会分成 A、B 两个小组，到不同的房间讨论。
每轮你们要决定：选'红'还是选'黑'。

得分规则很简单——
如果你们两组都选黑：各得 3 分（合作）
如果都选红：各扣 3 分（对抗）
如果一方红、一方黑：选红的得 5 分，选黑的扣 5 分

每轮讨论 3 分钟，然后同时公布选择。
中间可以派代表去对方房间'谈判'——说什么都可以，但没有约束力。

大家听懂了吗？"
```

### 教练的关键角色

| 阶段 | 教练在做什么 |
|------|-------------|
| **分组时** | 确保两组实力均衡（不要一边强一边弱） |
| **各房间内** | 教练轮流进入房间**旁听但不参与**，记录典型对话 |
| **谈判环节** | 监控谈判过程，防止"私下交易"破坏游戏设计 |
| **公布得分时** | **只报数字，不做评价**，让结果自己说话 |
| **复盘时** | 这是核心——引导深度讨论，不要急于给答案 |

### 关键把控

| 情况 | 处理方式 |
|------|---------|
| 两组很快建立信任（连续选黑） | 不要干预，让剧情自然发展；通常第 3–4 轮会出现"背叛" |
| 两组陷入对抗（连续选红） | 复盘时讨论："为什么我们无法走出对抗？" |
| 有人质疑游戏公平性 | 不要在过程中解释，等复盘时讨论："你感受到的不公平，在真实工作中存在吗？" |
| 谈判变成"签合同" | 允许，但每轮前提醒："上一轮的约定，这一轮还有效吗？" |

## 4. 复盘问题（4F 模型 · 深度版）

### Facts（事实）

> "我们一共进行了 __ 轮。两组的最终得分是多少？"
>
> "哪一轮是最重要的转折点？发生了什么？"
>
> "谈判的时候，对方说了什么？你们相信了吗？"

### Feelings（感受）

> "当你们发现被对方'背叛'（他们选红、你们选黑）的时候，感受是什么？"
>
> "当你们自己决定'背叛'对方的时候，感受又是什么？"
>
> "有没有人在过程中感到矛盾——既想合作，又想竞争？"

### Findings（发现）

> "这个游戏里，'红'代表什么？'黑'又代表什么？（在真实工作中）"
>
> "我们两组之间，有没有建立起信任？是什么帮助或阻碍了信任的建立？"
>
> "谈判的时候，我们说的话和我们实际上要做的事，是一致的吗？"
>
> "如果把这个游戏看成两个部门之间的关系，大家看到了什么？"

### Future（行动）

> "回到工作中，当我们和'对方部门/团队'打交道的时候，
> 今天的体验能给我们什么启发？"
>
> "如果我们想建立跨部门信任，今天学到了什么可以做的事？"

## 5. 变体适配

### 变体一：透明版（降低难度）

**规则**：两组在**同一空间**，但分别讨论，公布选择时所有人都能看到。

**适用场景**：初次接触这类游戏的团队，或时间较短（30 min 内）。

---

### 变体二：沟通版（增加复杂度）

**规则**：每轮允许两组**各发一条书面信息**（不能超过 20 个字），由教练转交。

**适用场景**：模拟"有限沟通渠道"下的信任建立。

---

### 变体三：教练介入版

**规则**：进行 3 轮后，教练可以匿名给其中一组一条"情报"（真或假）。

**适用场景**：模拟"信息不对称"对信任的影响。

---

### 人数调整

| 人数 | 调整方式 |
|------|---------|
| **12–20 人** | 标准玩法，每组 6–10 人 |
| **21–40 人** | 标准玩法，每组 10–20 人 |
| **> 40 人** | 分成四个组（A/B/C/D），用更复杂的得分矩阵 |

## 6. 物资清单

### 必备

- [ ] 两个房间（或同一房间用屏风/距离隔开）
- [ ] 得分矩阵（PPT 或白板，每轮更新）
- [ ] 计时器
- [ ] 纸和笔（每组讨论时记录观点）

### 可选

- [ ] 谈判记录表（记录每轮谈判内容，复盘时用）
- [ ] 观察员（每组安排 1 人专门记录讨论过程，不参与决策）

### 教练准备

- [ ] **提前演练得分计算**，确保现场不出错
- [ ] **准备复盘框架**（这个游戏的复盘是核心价值，教练必须充分准备）
- [ ] 如有可能，**录制各房间讨论过程**（征得同意后），复盘时回放精彩片段

### 准备时间

- 20–30 分钟（确认场地、准备得分矩阵、分组方案）
